try - catch - finally

function imaginarySquareRoot(number) {
 'use strict'; // Produces more exceptions and warnings and prohibits the use of some deprecated features
 let answer;
 try { // Throws any error to the catch block
  answer = String(squareRoot(number));
 } catch (error) { // Catches the error
  answer = squareRoot(-number) + "i";
 } finally { // Executes anyway
  return `+ or - ${answer}`;
 }
}

Questions

Classes vs Factories

Since classes are just functions under the hood, do we really need to use them? Should we?

Factories seem to be more robust against bugs due to avoiding the this keyword, and don't allow modifications of the objects/methods from the outside. It is a tiny bit slower though, but readability over speed seems to be a good compromise.
